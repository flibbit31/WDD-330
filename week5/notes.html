    Exceptions and warnings won’t always show up in browsers. ‘use strict’ on first line of a .js file or function. Modules automatically use strict mode. Linting tools are available to check for style. Check if objects and methods exist before using them if you’re unsure whether a browser feature exists or use Modernizr. Try using console.trace() when debugging. The debugger; command creates a breakpoint. Use the Error class can be used to create an error. You can throw anything, but you should only throw error objects. Try, catch, and finally are available in JavaScript. Test driven development: 1. Write tests (that initially fail) 2. Write code to pass the tests 3. Refactor the code 4. Test refactored code 5.
